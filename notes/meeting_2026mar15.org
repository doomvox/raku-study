**** METHODOLOGY
***** Small sample size: 19 developers
****** Expert programmers, working on open source projects on github
***** But ~250 programming tasks
****** Actual tasks, not contrived exercises
****** Real working conditions: interruptions allowed.
***** Developers are paid: $150/hr
***** Screens recorded while working
***** Developers estimate time twice: with or without AI use
***** Random "coin flip" to choose which tasks will be AI-allowed
***** They then ask developers how long they think it took

***** My comments:
****** studying expert programmers (they've got some funding)
******* much easier to study students
****** good to have a study at all
******* there aren't many studies like this of programming methodology
****** the programmers don't appear to do contracting
******* they'd have their own time-tracking software/habits
****** programmer time estimates suck
******* but this study demonstrates they suck more with the AI option
******* undercuts the anecdotal claims that "vibe" coding rules 
******* if you did studies like this on any latest bright and shiney, expect similar results
******** arguably this is hard on "AI"-- need to compare it to other fads
****** Note: they pay out on hours-- but the data they're measuring is hours
******* potentially contaminates the result (though I don't imagine that's happening here)
******* there's a slogan: when a metric becomes a target it stops working as a metric
****** Predictions
******* This study (and it's follow-ons, if any) will have little real world effect
******** hype and money are impervious to data
******* Expect to see criticism that they were Doing Something Wrong (if the study isn't just ignored)
******** e.g. using stale tooling rather than some new thing released last month
